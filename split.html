Split

A visualized calculator for spliting bills.

  1. Home
  2. UI/UX
  3. Split

A Personal Project | Tools: Proto.io, html, css, javascript, jQuery

A calculator acts as a straightforward tool to help people solve math problems. While even with the help of a calculator, splitting bills is sometimes still a complicated process when there are many people and items involved.

Design Process

The concept focuses on not only making the app a useful math tool but also visualizing the process in a way that can be easily understood. It is also important to simplify the process to create a smooth user experience.

Mapping

StoryBoard

Jack is having dinner with his friends in a restaurant. The food is good, everything is perfect, but...

When it comes to paying the bills, the calculation seems to be complicated.

Jack types people's names and prices of the dishes into the app.

The calculations are done with simple steps.

Experiments

Continuing the scenario in the storyboard, I proposed a math problem for experiment. I test with different existing apps to analyze the process for users.

Take Away from Experiments

Split is an app that helps users simplify and visualize the process of splitting a bill.

Instead of being overloaded by too many functions (e.g. sign in, search, add friends, create groups), I want to make this app a lightweight tool so that it can be easily understood and used.

Easy as 1. 2. 3

The user interface is divided into three simple steps: choosing how many people, adding items (by taking a photo or manually), and showing results. Without the steps for creating an account and signing in, the user flow is simplified, which makes it practical and easy to use.

1. Choose
2. Add
3. Result

Insights and Moving Forward

Applying simplicity

It is very easy to keep adding when designers start to brainstorm. So through this project, I learned how to stick to the original goal, clairify the needs and simplify the user flow.

Being willing to learn.

As design becomes more and more integrated into other fields, it is helpful to understand development so that design can be better implemented.
