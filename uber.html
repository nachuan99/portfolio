Uber

Redesign UI for multiple destinations.

  1. Home
  2. UI/UX
  3. Uber

Design Challenge | Tools: Sketch, Illustrator, Photoshop

Uber announced it added a multiple stops feature in October 2017. It becomes convenient for pick up/ drop other riders at multiple stops. While from a user's perspective, I find there might be some ways to improve the UI so that it is more noticeable for new users and more intuitive to use.

StoryBoard

Jack is having dinner with his friends in a restaurant. The food is good, everything is perfect...

After the dinner, they are thinking about how to go back home. Someone proposes taking Uber.

They live in different neighborhoods, and they are not familiar with how to add multiple stops in Uber...

Jack wonders if the user interface can be more intuitive to users.

Identify Problems

Problem 1: Create smooth user experience

In the current version, you have to go to a new page for multiple stops schedule. I explored several ways to integrate this feature in a smoother way so that you don't need to jump to another page.

Problem 2: Add extra riders

Uber enables users to split fair by swiping up from the bottom of the app and opening a new page. I want to integrate the splitting option into the multiple stops interface, since if there are multiple riders, it is likely they want to split.

Problem 3: Customize the mapping for multiple stops

Final Design

The final design integrates all the explorations above: a smooth user experience to add destinations, options to add riders and to split fare, and clear schedule of arrival time at all the stops.

Insights and Moving Forward

Considering all the details.

Through the project, I learned how to carefully think through details and consider their impact on users. To achieve clarity and simplicity, all the details have to fit into the user flow.

“Get rid of everything that is not essential to making a point.”

Christoph Niemann

Ask myself "why" and "so what".

The importance of asking “why” and "so what" is another lesson I learned. When I ask myself "why", I trace back to the reason; when I ask myself "so what", I think logically about the result. Asking me these questions constantly reminding me to match design with the goal of the project.
